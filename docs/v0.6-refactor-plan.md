# LEGEND v0.6 Refactor Plan

## Goal
Move LEGEND out of one giant `index.html` and into a safer structure while improving the first-time player experience.

## Why v0.6 matters
The current single-file build is useful for fast prototypes, but it is now risky to keep growing. Systems like combat, exploration, NPCs, saves, accessibility, and menus are all tangled together.

## Target structure

```text
index.html
css/style.css
js/data.js
js/storage.js
js/ui.js
js/game.js
```

## v0.6 design priorities

### 1. Progressive menu unlocks
New players should not see every town system immediately.

Opening Ashmere menu:
- Explore The First Road
- People of Ashmere
- Character Sheet
- Settings / Accessibility
- Save / Exit

Unlocks:
- Talk to Mara: Archive Hall, Ledger Hall, Save Vault
- First road event: Ashmere Inn, Trading Post
- First Road Token or first enemy defeat: Weapon Shop, Armor Shop
- First skill/resource event: The Commons
- Complete First Road: Hollow Keep teaser

### 2. Accessibility settings
Add persistent settings:
- Larger text
- Bolder text
- High contrast
- Reduced motion
- Simplified ASCII
- Spacious UI
- Readable font

Implementation idea:
- Save settings under `legend-settings-v1`
- Apply settings as body classes
- Keep Settings visible from the beginning

### 3. Save portability
Use `save.html` as the bridge system for now.

Future account/cloud saves can come later with Supabase, Firebase, PlayFab, or another backend.

### 4. QA updates
The browser QA runner should test:
- opening menu only shows early options
- locked/unlocked progression works
- Settings page opens
- accessibility toggles persist
- Save Vault page loads

## Refactor order
1. Create `css/style.css` from current style block.
2. Create `js/data.js` for classes, origins, items, enemies, events.
3. Create `js/storage.js` for save/load/settings/export helpers.
4. Create `js/ui.js` for rendering helpers.
5. Create `js/game.js` for game flow.
6. Replace `index.html` with a small loader.
7. Update `playtest.html` for v0.6 unlock checks.

## Safety note
This should happen in small commits so the playable build is not accidentally broken.
