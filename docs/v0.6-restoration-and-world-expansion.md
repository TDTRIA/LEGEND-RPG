# LEGEND v0.6 Restoration and World Expansion Plan

## Purpose
v0.6 gave LEGEND a cleaner foundation, but some v0.5.1 depth was intentionally reduced during the refactor. This document defines the next restoration and expansion phase.

## Core goals

1. Restore systems trimmed during the v0.6 refactor.
2. Further reduce early menu overwhelm.
3. Hide unusable menu options instead of showing too many locked cards.
4. Make towns discoverable through exploration.
5. Improve ASCII scene art and give NPCs unique portraits.
6. Add Roadwarden-inspired travel, time, survival, mapping, and consequence systems.
7. Research Baladins more carefully before copying any specific ideas.

---

## 1. Restore trimmed v0.5.1 systems

### Restore / improve
- Enemy intent text during combat.
- Momentum meter and class moves.
- Guard as a more meaningful combat choice.
- Deeper Old Road event variety.
- Road Token progress guarantees.
- Bell Shard pacing.
- Mastery rewards from skills.
- Camp Supplies as route survival tools.
- More complete NPC dialogue topics.
- Brenn turn-ins and better quest completion feedback.
- Save Vault access from town once unlocked.

### Priority order
1. Combat depth: intent, guard, class moves, momentum.
2. Exploration depth: more route events, guaranteed Road Token pacing.
3. Town depth: NPC topics, better shops, deeper Ledger/Archive.
4. Skill depth: mastery rewards and event bonuses.

## 2. Condense the menu further

Current v0.6 shows locked cards. That is better than everything being open, but still visually noisy.

### New rule
If a system cannot be used yet, hide it unless it is narratively important.

### Starter Ashmere menu should only show
- Explore The First Road
- People of Ashmere
- Character Sheet
- Settings / Accessibility
- Save / Exit

### Hidden until discovered/unlocked
- Archive Hall
- Ledger Hall
- Ashmere Inn
- Trading Post
- Weapon Shop
- Armor Shop
- The Commons
- Save Vault
- Hollow Keep

### Exceptions
Some locked items may appear as a single combined hint card:

> More of Ashmere will open as you meet people and survive the road.

This keeps mystery without overwhelming the player.

## 3. Town discovery system

Instead of every town existing from the beginning, towns should be found one at a time through exploration.

### New data model idea
```js
world: {
  currentTown: 'ashmere',
  discoveredTowns: ['ashmere'],
  knownRoutes: ['oldRoad'],
  mapProgress: 0
}
```

### Town discovery flow
1. Start in Ashmere.
2. Explore Old Road.
3. Random events can reveal landmarks, signposts, travelers, maps, or rumors.
4. After enough discovery progress, a new town is found.
5. New town unlocks its own NPCs, shops, risks, and routes.

### Example towns

#### Ashmere
First town. Archive, Ledger, Mara, Brenn, basic survival.

#### Briarfall
A wet forest settlement with hunters, herbalists, and hidden paths.

#### Cindervale
A mining town near old furnace ruins. Smithing, ore, heat, smoke, labor politics.

#### Greywake Crossing
A river/trade town. Ferries, tolls, travelers, rumors, smuggling.

#### Hollowmere
A half-abandoned town near the Hollow Keep. Late early-game dread.

## 4. Roadwarden-inspired systems

Roadwarden is useful inspiration because it combines travel, survival, mapping, settlements, and prose-based choices.

### Concepts to adapt for LEGEND

#### Time of day
Add a simple day clock:
- Dawn
- Midday
- Dusk
- Night

Exploring advances time. Night increases risk.

#### Travel fatigue
Add road weariness:
- Low fatigue: normal checks.
- Medium fatigue: harder checks.
- High fatigue: more danger and worse outcomes.

#### Appearance / condition
NPC reactions can change based on condition:
- Clean / rested
- Road-worn
- Injured
- Bloodied
- Strange aura

#### Mapping
Exploration reveals routes, landmarks, and towns.

#### Prose-based odds
Instead of always showing exact percentages, describe risk:
- This looks safe.
- This could go badly.
- This is reckless.
- This feels almost impossible.

#### Role-based choices
Class, origin, and keepsake unlock unique options.

Examples:
- Ranger notices a hidden trail.
- Mage senses a runic scar.
- Thief spots a fake lock.
- Wandering Cook recognizes edible roots.
- Archivist's Child understands old town records.

## 5. Baladins research placeholder

Baladins was mentioned as inspiration, but we should verify the exact systems before adapting anything directly.

Questions to research:
- What makes its quest structure interesting?
- Does it use time loops, party roles, town events, or board-game style turns?
- How does it make short sessions feel meaningful?
- Which ideas fit a solo browser text RPG?

Until verified, do not implement specific Baladins mechanics by memory.
