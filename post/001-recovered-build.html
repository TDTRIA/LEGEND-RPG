Devlog 001 / Recovered Build Archive

I Recovered My First Game

Legend started as the first game I ever came up with: a fantasy RPG written entirely in Windows Batch. It was rough, strange, menu-driven, and full of placeholder ideas, but it already had the bones of a real RPG.

The surviving file appears to be LegendTFC release 1.0. It had account creation, saving, combat, shops, inventory, armor, weapons, a bank, skills, quests, monster drops, and a Stronghold boss named Lacoyx.

I remember later versions existing. I remember classes, new skills, and even holiday events. Those versions seem to be gone now, but that ended up giving the remake its identity.

The Remake Idea

Instead of pretending the missing versions never existed, the remake treats them as part of the story. Legend is now a recovered world. Some systems survived. Some were lost. Some are slowly being restored.

Varrock, RealmBank, the Quest Hall, the Skill Plot, and the Stronghold are no longer just menus. They are pieces of an old game world trying to remember what they used to be.

Recovered Build v0.2

The current browser version rebuilds Legend as an HTML5 game with a new interface, class selection, a Quest Hall, recovered memories, a better battle screen, and the first version of the Stronghold encounter.

It is still early, but the goal is clear: preserve the spirit of the original batch game and build the version it could have become.

What Comes Next

The next major update is planned around exploration. Instead of simply grinding battles, the player will be able to explore roads, ruins, campsites, forests, and strange events outside Varrock. Choices will matter. Random encounters will feel closer to a tabletop RPG, with risk, reward, and class-based checks.

The Trading Post and Smuggler are also planned so players can sell enemy drops, make money, and interact with the world in more interesting ways.
